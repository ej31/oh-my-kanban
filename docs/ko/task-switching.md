# 태스크 전환 (Task Switching)

작업 중 다른 태스크로 전환해야 할 때, 기존 작업을 보류하고 새 작업을 시작할 수 있습니다.

## 개요

`omk hooks switch-task` 명령으로 현재 세션의 Work Item을 보류 상태로 이동하고, 새 태스크를 위한 준비를 합니다. 전환 이력은 타임라인에 기록됩니다.

## CLI 명령

### 태스크 전환

```bash
# 기본 전환 (기존 WI를 보류 상태로 이동)
omk hooks switch-task

# 새 태스크 제목과 이유 지정
omk hooks switch-task --new "긴급 버그 수정" --reason "프로덕션 장애"

# 특정 세션 지정
omk hooks switch-task --session-id abc12345 --new "새 기능"
```

## 동작 방식

1. **기존 Work Item 보류**: 현재 세션의 `work_item_ids`가 `stale_work_item_ids`로 이동합니다.
2. **타임라인 기록**: `task_switch` 타입의 이벤트가 추가됩니다.
3. **세션 저장**: 변경된 상태가 로컬 파일에 저장됩니다.

## 전환 이후

- 보류된 Work Item은 삭제되지 않습니다. Plane에서 계속 조회할 수 있습니다.
- `--new` 옵션으로 새 태스크 제목을 지정하면, 다음 세션 훅 실행 시 자동으로 새 WI가 생성됩니다.
- `/oh-my-kanban:focus` 스킬로 다른 기존 WI에 포커스를 맞출 수도 있습니다.

## 타임라인 예시

```
2026-01-01T10:00:00 [scope_init] 세션 시작
2026-01-01T10:30:00 [task_switch] 태스크 전환 | 새 태스크: 긴급 버그 수정 | 이유: 프로덕션 장애
2026-01-01T11:00:00 [prompt] 세션 정상 종료
```

## 주의사항

- 활성 세션이 없으면 전환할 수 없습니다.
- 전환 시 기존 세션의 범위(scope)와 통계(stats)는 유지됩니다.
- 빈번한 태스크 전환은 세션 추적의 정확도를 낮출 수 있습니다.
