# omk-analyze-lv3: 심층 아키텍처 분석

아키텍처 패턴, 데이터 흐름, 품질 지표를 심층 분석합니다.

## 목적

- 아키텍처 패턴 식별 (MVC, 레이어드, 이벤트 드리븐 등)
- 데이터 흐름 추적 (입력 → 처리 → 출력)
- 결합도/응집도 평가
- 잠재적 위험 요소 식별
- 기술 부채 탐지

## 추천 모델

**Opus** - 심층 추론에 최적화

## 언제 사용하나요?

- 대규모 리팩토링을 계획할 때
- 아키텍처 결정을 내려야 할 때
- 보안 검토를 수행하기 전
- 성능 병목을 찾아야 할 때

## 수행 지침

1. **아키텍처 패턴 식별**
   - 전체 디렉토리 구조에서 아키텍처 레이어 식별
   - 각 레이어의 경계와 책임 범위 분석
   - 레이어 간 통신 방식 확인

2. **데이터 흐름 추적**
   - 주요 유스케이스 3개를 선정
   - 각 유스케이스의 데이터 흐름을 입력부터 출력까지 추적
   - 데이터 변환 지점 식별

3. **결합도/응집도 평가**
   - 모듈 간 결합도 (import 관계, 공유 상태)
   - 모듈 내 응집도 (관련 기능이 한 곳에 모여 있는지)
   - 순환 의존성 탐지

4. **잠재 위험 식별**
   - 보안 취약점 (입력 검증 누락, 인증/인가 미비)
   - 성능 병목 (N+1 쿼리, 불필요한 직렬화)
   - 확장성 제약 (하드코딩된 제한, 단일 장애점)
   - 테스트 커버리지 빈틈

5. **출력 형식**
   ```
   === 심층 아키텍처 분석 ===

   [아키텍처 패턴]
   유형: {패턴 이름}
   레이어: {레이어 목록}
   평가: {강점과 약점}

   [데이터 흐름 - 유스케이스: {이름}]
   1. {시작점} → {처리1} → {처리2} → {끝점}
   변환: {어디서 어떻게 변환되는지}

   [품질 지표]
   결합도: {높음/중간/낮음} — {근거}
   응집도: {높음/중간/낮음} — {근거}
   순환 의존성: {있음/없음} — {해당 모듈}

   [잠재 위험]
   - [심각도: 높음] {위험 설명} ({위치})
   - [심각도: 중간] {위험 설명} ({위치})

   [권장 사항]
   1. {단기 개선 사항}
   2. {중기 개선 사항}
   3. {장기 개선 사항}
   ```
